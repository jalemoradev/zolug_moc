Tokens — tema oscuro

Regla dura — estos son los colores del modo oscuro. Los tokens de arriba son la paleta oscura de zolug (medida de Plasma, WCAG AA verificado). Toda pantalla de módulo toma sus colores oscuros de estos tokens exactos, a través de las variables del shell layout/dashboard.html —--bg-page (#0d0c13), --bg-surface (#14121c), --bg-elevated (#1a1922), --text-primary (#ffffff), --text-muted (#a2a9ad), --accent-solid (#5a58c2), --accent-on-dark (#7474ef), --logo (#955dcd), --error (#df5158)— más --border y --hair (ver Bordes). Prohibido inventar, hardcodear o sustituir un color oscuro fuera de esta paleta.

Regla dura: ningún texto por debajo de 14px; el cuerpo de lectura es 16px. Prohibido inventar tamaños fuera de estos 6 tokens. WCAG no fija un mínimo de píxeles —lo que exige es que el texto se pueda ampliar al 200% sin romperse (SC 1.4.4)— pero el piso de 14px y el cuerpo de 16px son la política del proyecto para baja visión, presbicia y astigmatismo. Estos tamaños no resuelven daltonismo: eso es color, no tamaño. Tamaños iguales o mayores a 18px (o 14px en negrita) cuentan como “texto grande” y bajan su umbral de contraste a 3:1.

Cómo se manipula el color: los tres SVG traen el color incrustado como .cls-1{fill:#0d0c13}. Ese valor es exactamente bg/page, así que sobre el tema oscuro el logo queda invisible tal cual viene. Para controlarlo desde el código, reemplace ese fill por currentColor: el SVG heredará el color CSS de su contenedor (aquí se ve en blanco, en acento y en su color original sobre claro). Como componente React se importa el SVG y se colorea con la clase de texto (p. ej. text-white o el token de acento). Los archivos fuente no se han modificado — si quiere, los convierto a currentColor de una vez.

Componentes — sistema del proyecto

shadcn/ui

ui.shadcn.com

Este proyecto usa shadcn/ui como sistema de componentes. No es una dependencia que se instala: son componentes que se copian al repositorio (sobre Radix UI + Tailwind), así que el equipo controla y adapta cada uno.

  • Encaja con estos tokens: shadcn se tesea con variables CSS — las --color-* de esta guía se mapean directo a su tema (background, foreground, primary, border…).
  • Accesible por dentro: se apoya en Radix UI (primitivas headless), lo que sostiene el objetivo WCAG AA.
  • Mismo stack: Tailwind + React + Vite, sin capas de estilo que peleen con la paleta.

Pendiente al configurar el proyecto: mapear los tokens de esta guía al tema de shadcn. No se hace aquí — este documento es la referencia visual, no la configuración.

Tokens en CSS — listos para Tailwind / React + Vite

  
Pendiente, sin inventar: los colores de gráficas y las píldoras de estado (verde / rojo / amarillo / naranja) de Plasma no se pudieron medir con precisión desde las capturas (antialiasing) y no hay acceso al CSS de la app. Se definirán con valores exactos cuando se conozcan. accent/logo-ref es el color real del logo de Plasma; se incluye como referencia pero no cumple AA como texto (3.92:1 sobre bg/elevated), por eso la interfaz usa accent/on-dark.

  
Cómo se miden estos ratios: siempre contra el peor fondo de cada tema — bg/elevated en oscuro, bg/page en claro. Medir solo contra el fondo de página oculta fallas sobre paneles y tarjetas, que es justo donde vive la mayoría del texto. Verificado con la fórmula normativa de WCAG 2.1 (umbral de linealización 0.04045, vigente desde mayo 2021).
